

# Plan: Google OAuth Registration + Login + Account Linking

## Summary
1. **Registration**: Replace email/password signup with mandatory Google OAuth. After Google sign-in, if the user has no profile yet (new user), show a form to complete required info (MSSV, full name, etc.) before account is created.
2. **Login**: Add a "Đăng nhập bằng Google" button alongside existing MSSV+password login.
3. **Personal Info page**: Show Google account link status with ability to link/unlink/change Google account.

## Technical Approach

### Step 1: Configure Social Auth (Google)
- Use the Lovable Cloud managed Google OAuth (no API key needed).
- Call the Configure Social Login tool to generate `src/integrations/lovable/` module.
- Use `lovable.auth.signInWithOAuth("google", { redirect_uri })` for all Google auth flows.

### Step 2: Update `handle_new_user` trigger
- The existing trigger creates a profile on signup. Google OAuth users will also trigger this. The `raw_user_meta_data` will contain Google info (name, email, avatar).
- Update the trigger to also store `avatar_url` from Google metadata if available.

### Step 3: Redesign Registration Flow (`MemberAuthForm.tsx`)
- Replace the current registration form with a **two-step process**:
  - **Step 1**: Click "Đăng ký bằng Google" button → Google OAuth redirect.
  - **Step 2**: After returning from Google, if user is new (profile not complete / no `student_id`), show a **completion form** asking for: MSSV, full name (pre-filled from Google), policy agreement.
- Remove email/password fields from registration entirely.
- The completion form updates the `profiles` table with the additional info.

### Step 4: Add Google Login Button (`MemberAuthForm.tsx`)
- Add a "Đăng nhập bằng Google" button in the login tab, styled prominently with Google icon.
- Uses `lovable.auth.signInWithOAuth("google", { redirect_uri: window.location.origin })`.
- Existing MSSV + password login remains as alternative.

### Step 5: Handle OAuth Callback in `AuthContext.tsx`
- On `SIGNED_IN` event, check if the user's profile has `student_id` filled.
- If not (new Google user), set a flag to show the profile completion form.
- Add a new state `needsProfileCompletion` to context.

### Step 6: Google Account Linking in `PersonalInfo.tsx`
- Add a new card/section "Tài khoản liên kết" (Linked Accounts).
- Check `user.identities` from Supabase auth to see if Google identity exists.
- If not linked: Show "Liên kết Google" button that calls `lovable.auth.signInWithOAuth("google", ...)` with `linkIdentity` or similar.
- If linked: Show the linked Google email with option to change/unlink.
- Use `supabase.auth.linkIdentity({ provider: 'google' })` for linking and `supabase.auth.unlinkIdentity()` for unlinking.

### Files to Modify
1. **`src/integrations/lovable/`** — Auto-generated by Configure Social Login tool
2. **`src/components/MemberAuthForm.tsx`** — Major rewrite: add Google buttons, two-step registration
3. **`src/contexts/AuthContext.tsx`** — Add `needsProfileCompletion` state, handle Google OAuth users
4. **`src/pages/PersonalInfo.tsx`** — Add linked accounts section
5. **Database migration** — Update `handle_new_user()` trigger to handle Google OAuth metadata (avatar_url from Google)

### Database Changes
- Update `handle_new_user()` function to extract `avatar_url` from `raw_user_meta_data` for Google users.
- No new tables needed.
